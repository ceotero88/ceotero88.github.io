Healthcare Appointment No-Show Analysis

A tool for analyzing patient no-show trends and demographic factors influencing missed medical appointments.

View Patient No-Show Analysis

The Patient No-Show Analysis is a visualization interface designed to help healthcare providers understand factors influencing patient no-shows to appointments. Through interactive visualizations, this app allows providers to explore demographic trends and health-related factors associated with higher no-show rates, enabling data-driven strategies for targeted interventions.

Key visualization approaches include a bar chart comparing no-show rates across health conditions such as hypertension and diabetes, a line plot revealing trends in no-show rates by month to identify seasonal patterns, and an interactive line plot showing how age and gender distributions relate to no-show rates. This application empowers healthcare providers with actionable insights to optimize appointment scheduling, resource allocation, and patient engagement strategies.

Sentiment Analysis of Amazon Electronics Reviews: Insights for E-Commerce

A research paper examining customer satisfaction and behavior in electronics reviews, using sentiment analysis to uncover key insights for e-commerce strategies.

View Research Paper

This research paper investigates customer satisfaction for electronics sold on Amazon, leveraging sentiment analysis and Natural Language Processing (NLP) techniques to uncover consumer behavior patterns. Using the 2020 Amazon Product Reviews Dataset, the analysis identifies common complaints and compliments, evaluates time-based trends, and examines differences between purchases on Amazon and brand-specific stores. The study also highlights the importance of review length and sentiment, revealing how short reviews often capture concise feedback while longer reviews provide deeper insights.

Conducted using Python on Google Colab, this project utilized libraries like NLTK and TextBlob for sentiment scoring and data visualization tools to identify trends. The findings offer actionable recommendations for manufacturers and retailers, such as improving product features based on customer feedback and refining marketing strategies to emphasize highly-rated attributes. Overall, the paper underscores the value of sentiment analysis in understanding consumer preferences and enhancing the e-commerce experience.

Arduino Sound Sensor LED Control

A circuit designed to change an LED's color based on sound levels using an Arduino, sound sensor, and RGB LED.

This project involved designing and building a circuit using an Arduino Uno to control the color of an RGB LED based on sound levels picked up by a sound sensor. The setup included a breadboard for organizing connections, jumper cables to link components, resistors to protect the circuit, and an RGB LED that changed colors depending on the input from the sensor.

The sound sensor captured ambient noise levels and sent voltage signals to the Arduino. I programmed the Arduino in C++ using the Arduino IDE, mapping the sensor’s analog input to RGB color values. Functions like analogRead() gathered the sound data, while analogWrite() adjusted the red, green, and blue pins on the LED, making it respond dynamically to sound in real-time.

To ensure the circuit worked smoothly, I added resistors to prevent damage to the LED and made sure all jumper connections were solid. The final project combined hardware and software to create a responsive system that translated sound into light, offering hands-on experience with signal processing and embedded systems.
